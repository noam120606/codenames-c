◆ InputConfig

typedef struct InputConfig InputConfig

Configuration pour créer un Input. Tous les champs ont des valeurs par défaut définies via input_config_init. La structure peut être modifiée à tout moment et passée à input_create. Note : Certains champs sont gérés en interne par l'input (text, cursor_pos, textures, etc.) et ne doivent pas être modifiés directement après la création de l'input. Seuls les champs "configurables" (x, y, w, h, font_path, etc.) peuvent être modifiés pour changer l'apparence ou le comportement de l'input.

Paramètres
xOffset horizontal relatif au centre de la fenêtre (pixels). Positif vers la droite, négatif vers la gauche.
yOffset vertical relatif au centre de la fenêtre (pixels). Positif vers le haut, négatif vers le bas. Si x = 0 et y = 0, le centre de l'input est au centre de la fenêtre. Formules appliquées dans input_create : screen_x = WIN_WIDTH/2 + x - w/2 screen_y = WIN_HEIGHT/2 - y - h/2
wLargeur de l'input.
hHauteur de l'input.
font_pathChemin vers la police à utiliser pour le texte de l'input.
font_sizeTaille de la police.
placeholdersTableau de chaînes de caractères utilisées comme placeholders cycliques lorsque l'input est vide. Peut être NULL si aucun placeholder n'est utilisé.
placeholder_countNombre de placeholders dans le tableau placeholders.
submitted_labelTexte à afficher avant le texte soumis (ex: "Pseudo : "). Peut être NULL pour ne rien afficher.
maxlenLongueur maximale du texte saisi. Si le texte dépasse cette longueur, il sera tronqué.
bg_colorCouleur de fond de l'input (utilisée si bg_path est NULL).
border_colorCouleur de la bordure de l'input.
text_colorCouleur du texte saisi.
paddingPadding intérieur de l'input (espace entre le bord et le texte).
centeredSi 1, le texte/placeholder et le curseur sont centrés horizontalement. Si 0, ils sont alignés à gauche.
bg_pathChemin vers l'image de fond de l'input. Si NULL, un fond uni de couleur bg_color est utilisé. L'image est chargée automatiquement par input_create et doit être libérée lors de la destruction de l'input.
bg_paddingPadding à appliquer avec l'image de fond (>= 0). Ignoré si bg_path est NULL. Si -1, le padding est calculé automatiquement pour centrer le contenu dans l'image de fond.
allowed_patternRegex POSIX étendue appliquée à chaque caractère saisi. Si NULL, tout caractère est accepté. Ex: "^[A-Za-z0-9]$"
submit_patternRegex POSIX étendue appliquée au texte complet au moment de la soumission. Si définie et non satisfaite, la soumission est ignorée. Ex: "^[0-9]{5}$"
init_textTexte initial à afficher dans l'input. Si NULL ou chaîne vide, l'input commence vide.
on_submitFonction de rappel à appeler lorsque l'input est soumis (par exemple, en appuyant sur Entrée). La fonction doit prendre un SDL_Context* (contexte SDL) et un const char* (le texte soumis) et retourner void. Si NULL, aucune fonction ne sera appelée lors de la soumission.

◆ input_create()

Input * input_create ( SDL_Renderer * renderer,
InputId id,
const InputConfig * cfg )

Crée un nouvel input à partir d'une configuration. Si cfg est NULL, les valeurs par défaut sont utilisées. L'input créé doit être détruit avec input_destroy pour éviter les fuites de mémoire.

Paramètres
rendererRenderer SDL utilisé pour charger la texture de fond si bg_path est défini dans la configuration.
idIdentifiant de l'input (ex: INPUT_ID_NAME). Peut être utilisé pour différencier les inputs dans les callbacks.
cfg_inPointeur vers la configuration de l'input. Si NULL, les valeurs par défaut sont utilisées. La configuration est copiée, donc le pointeur peut être libéré ou modifié après l'appel sans affecter l'input créé.
Renvoie
Pointeur vers l'input créé, ou NULL en cas d'erreur (par exemple, échec d'allocation mémoire ou de chargement de la texture de fond).
